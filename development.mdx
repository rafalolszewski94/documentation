## Contributing to IPBun Docs

Help improve the IPBun documentation by contributing to this repository.

<Info>
  **Prerequisites**:
  - Node.js version 19 or higher
  - Git for version control
  - A GitHub account
</Info>

## Local Development Setup

<Steps>
<Step title="Clone the repository">

```bash
git clone https://github.com/your-org/ipbun-docs.git
cd ipbun-docs
```
</Step>

<Step title="Install dependencies">

```bash
npm install
```
</Step>

<Step title="Install Mintlify CLI">

```bash
npm install -g mintlify
```
</Step>

<Step title="Start development server">

```bash
mint dev
```

A local preview will be available at `http://localhost:3000`.

</Step>
</Steps>

## Making Changes

### Documentation Updates

1. **API Changes**: Update the OpenAPI specification in `api-reference/openapi.json`
2. **Endpoint Documentation**: Modify files in `api-reference/endpoint/`
3. **General Content**: Update `.mdx` files in the appropriate directories

### Code Examples

When adding code examples, follow these guidelines:

- Include examples in multiple programming languages
- Use clear, readable code with comments
- Test all code examples to ensure they work
- Follow language-specific best practices

### Testing API Endpoints

Before documenting new endpoints, test them thoroughly:

```bash
# Test basic functionality
curl https://api.ipbun.com/api/v1/health

# Test with different formats
curl "https://api.ipbun.com/api/v1/ip?format=json"
curl "https://api.ipbun.com/api/v1/ip?format=jsonp&callback=myCallback"

# Test rate limiting
curl -I https://api.ipbun.com/api/v1/ip
# Check X-RateLimit-Remaining header
```

## Pull Request Process

1. **Fork** the repository
2. **Create** a feature branch: `git checkout -b feature/your-feature-name`
3. **Make** your changes and test locally
4. **Commit** your changes: `git commit -m "Add: description of changes"`
5. **Push** to your branch: `git push origin feature/your-feature-name`
6. **Create** a Pull Request with a clear description

## Code Quality

- Use clear, descriptive commit messages
- Follow MDX formatting guidelines
- Ensure all links are valid (`mint broken-links`)
- Test documentation builds locally before submitting

## Troubleshooting

<AccordionGroup>
  <Accordion title='Mintlify CLI Issues'>

    **Could not load the "sharp" module**: Upgrade to Node.js v19+ and reinstall the CLI:

    ```bash
    npm remove -g mintlify
    npm install -g mintlify
    ```
  </Accordion>

  <Accordion title="Documentation Build Errors">

    **Unknown error**: Clear Mintlify cache and restart:

    ```bash
    rm -rf ~/.mintlify
    mint dev
    ```
  </Accordion>

  <Accordion title="Link Validation">

    Check for broken links in your documentation:

    ```bash
    mint broken-links
    ```
  </Accordion>
</AccordionGroup>
