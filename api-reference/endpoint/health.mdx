---
title: 'Health Check'
openapi: 'GET /api/v1/health'
---

Returns the health status of the IPBun API service, including server status, timestamp, and version information.

## Response

Returns a JSON object with the current health status of the service.

```bash
curl https://api.ipbun.com/api/v1/health
```

```json
{
  "status": "ok",
  "timestamp": "2025-01-15T10:30:00.000Z",
  "version": "1.0.0"
}
```

## Response Fields

<ResponseField name="status" type="string" required>
  Health status of the service. Currently always returns "ok" when the service is operational.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp of when the health check was performed.
</ResponseField>

<ResponseField name="version" type="string" required>
  API version in semantic versioning format (e.g., "1.0.0").
</ResponseField>

## Usage

Use this endpoint to monitor the availability and status of the IPBun API service. A successful response (HTTP 200) with `"status": "ok"` indicates the service is operational.

```bash
# Check service health
if curl -s https://api.ipbun.com/api/v1/health | jq -e '.status == "ok"' > /dev/null; then
  echo "Service is healthy"
else
  echo "Service is unhealthy"
fi
```
